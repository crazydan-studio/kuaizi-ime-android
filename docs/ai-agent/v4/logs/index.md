# 开发日志索引

本目录存放开发日志文档。文档按年组织子目录，以 `{月}-{日}.md` 形式命名（月和日采用两位数字）。文档为历史记录，不对已有内容做更新。

---

## 日志组织规范

### 目录结构

```
logs/
├── index.md          ← 本文档
├── images/           ← 共享图片
├── 2026/
│   ├── 05-12.md      ← 2026年5月12日的日志
│   ├── 05-13.md      ← 2026年5月13日的日志
│   └── 06-01.md      ← 2026年6月1日的日志
└── ...
```

### 日志内容模版

```markdown
# {YYYY-MM-DD} 开发日志

## 今日工作

### {工作项 1}
- **关联计划**：{引用 plans/ 中的计划编号}
- **工作内容**：{描述做了什么}
- **关键决策**：{记录做出的重要决策及理由}
- **遇到的问题**：{记录遇到的问题和解决方式}

### {工作项 2}
...

## 明日计划

- {列出明日计划做的工作}

## 备注

{其他需要记录的信息}
```

### 规范说明

1. 每个工作日创建一个日志文件
2. 日志内容按「今日工作 → 明日计划 → 备注」的结构组织
3. 每个工作项必须关联到具体的计划编号
4. 关键决策需记录决策理由，便于后续回顾
5. 遇到的问题和解决方式需详细记录，为后续类似问题提供参考
6. 日志为历史记录，不对已有内容做更新
